# Курсовая работа по дискретной математике

## Как приступить к работе:

1. Открыть репозиторий-шаблон https://github.com/dismath-coursework/calculator-template
2. Нажать зеленую кнопку **Use this template** в правом верхнем углу, выбрать **Create a new repository**.
<p align="center">
  <img width="515" height="260" alt="Снимок экрана 2026-06-29 221102" src="https://github.com/user-attachments/assets/2d355a94-98ff-40c2-9190-0144b37a59ac" />
</p>

3. Заполнить форму создания репозитория:
    1) Поле **Owner** - организация `dismath-coursework`
    2) Поле **Repository name** в формате `номер группы_Фамилия-Имя` (например, для группы 5130201/30101 - `30101_Martynenko-Anna`)
    3) Поле **Visibility** обязательно поставить `Private`
4. Нажать **Create repository from template**

<p align="center">
  <img width="826" height="719" alt="Снимок экрана 2026-06-29 221831" src="https://github.com/user-attachments/assets/085cc4f8-16cf-4a55-a065-034b11445f45" />
</p>

5. Клонировать репозиторий на свою машину
6. в .github/workflows/test.yml вписать свой алфавит в `jobs: test-all: - name: Run hidden test suite with: alphabet`

7. Keep calm and enjoy coding :)

Наверное нужно что-то сказать про CMake (хотя бы о том, что свои файлы .cpp тоже надо добавлять)  или не надо..

## Как отправить на проверку и посмотреть результаты
1. **После ручного тестирования** закоммитьте свое решение и запушьте **в свой репозиторий**
2. Результаты можно посмотреть во вкладке **Actions** своего репозитория
3. Необходимый минимум - все пройденные тесты по операциям `add, subtract, multiply, division`
4. Приветствуются бонусные операции!

## Ограничения
- Автоматические проверки призваны помочь провалидировать финальные результаты, а не заменить ручное тестирование.
- Всего доступно 20 автоматических проверок (то есть 20 коммитов). Используйте их с умом!
- После дедлайна (будет указан позднее, предварительно - дата сдачи курсовой на сессии) автоматические проверки будут недоступны.
